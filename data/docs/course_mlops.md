## 과정 한눈에 보기

- 공식 과정명: AI Ready Data 기반 Cloud-Native 자동화를 위한 MLOps 엔지니어 양성과정
- 상담용 축약명: AI Ready 데이터 엔지니어링 캠프
- 서브타이틀: AI Ready Data 기반 Cloud-Native 자동화 시스템 설계
- 한 줄 정의: AI가 멈추지 않고 안정적으로 돌아가게 만드는 기술을 배우는 과정
- 핵심 질문: 이 모델을 어떻게 안정적으로 운영할 것인가
- 기간: 6개월 (960시간)
- 교육 장소: G밸리 캠퍼스 안내 기준
- 비용: 교육비 0원 + 훈련지원금 240만 원 안내 기준
- 운영 사업: 2026 K-디지털 트레이닝 AI 특화훈련 엔코아 캠퍼스

## 과정 핵심 요약

- 이 과정은 모델을 "만드는 것"보다 **실제로 안정적으로 운영되게 만드는 것**에 초점이 있습니다.
- 데이터 엔지니어링부터 모델 운영 자동화까지 한 흐름으로 연결해 배우는 통합형 과정입니다.
- 특히 `AI Ready Data`, `CI/CD/CT`, 자동 재학습, 장애 자동 복구, 무중단 배포 같은 운영 실무 역량을 강조합니다.

## 핵심 가치 3축

### 1. 데이터 엔지니어링 + MLOps 엔지니어 통합

- `Python`, `SQL`, `NoSQL` 기반 데이터 기본기부터 시작합니다.
- `Spark`, `Kafka`, `Docker`, `Kubernetes`, `MLflow`, `Airflow`까지 이어지는 흐름을 한 과정 안에서 확장합니다.
- 데이터 파이프라인, 모델 운영, 자동화를 따로 배우는 것이 아니라 하나의 포트폴리오로 누적합니다.

### 2. AI Ready Data

- 엔코아의 데이터 컨설팅 경험을 바탕으로, 레거시 데이터를 AI 학습에 적합한 형태로 변환하는 방법을 다룹니다.
- 표준화, 스키마 설계, 품질 기준, 메타데이터 정비 같은 **데이터 준비 역량**이 핵심입니다.
- 일반 데이터 엔지니어링 과정보다 AI 학습용 데이터 준비에 더 직접적으로 맞춘 교과 성격을 가집니다.

### 3. CI/CD/CT 기반 자동화 완성

- 모델이 한 번 배포되고 끝나는 것이 아니라, 데이터 변경 시 자동 재학습되고 장애 시 자동 복구되는 시스템을 지향합니다.
- 무중단 배포, 모델 생애주기 자동화, 운영 모니터링까지 연결된 구조를 학습합니다.
- 실제 서비스 운영 환경에 가까운 `MLOps 엔지니어` 역량을 목표로 합니다.

## 추천 대상

- 데이터 인프라, 시스템 운영, 자동화에 관심 있는 사람
- 모델 개발보다 운영 안정성, 배포, 재학습, 모니터링에 더 흥미가 있는 사람
- 데이터 파이프라인과 클라우드 인프라를 함께 다루는 역량을 쌓고 싶은 사람
- 비전공자라도 시스템 흐름과 자동화 구조를 배우며 AI 인프라 쪽으로 진입하고 싶은 사람

## 단계별 결과물

- Lv.1: 파이썬 기반 데이터 파이프라인, AI Ready Data 구축 프로젝트, 표준 사전·데이터 모델 포트폴리오
- Lv.2: `Spark`, `Kafka` 기반 분산 처리 시스템, 실시간 로그 통합 파이프라인, 데이터 레이크하우스
- Lv.3: `MLflow`, `Airflow`, `Docker`, `Kubernetes` 기반 모델 생애주기 자동화, 장애 자동 복구 시스템, 카나리 배포/무중단 배포 시스템

## 대표 실무 적용 사례

### 공공 데이터 관리체계

- 행정안전부의 `AI Ready` 공공데이터 관리체계 도입 흐름
- 고가치 공공데이터 개방과 데이터 포맷·메타데이터·품질 기준 정비

### 공공데이터 공통표준

- 공통표준용어, 공통표준단어, 공통표준도메인 기반의 표준화 흐름
- 레거시 데이터를 AI 학습용으로 변환하는 실무 배경

### 기업의 AI Ready Data 구축 흐름

- 현대자동차, 삼성, LG, 현대 등 기업의 데이터 인프라 투자 확대
- 대기업이 AI 경쟁력의 핵심을 "데이터 준비"에 두고 있다는 점이 강조됩니다.

## 핵심 기술 스택

- 프로그래밍/협업: `Python`, `pandas`, `Git/GitHub`, `ChatGPT`, `Codex Code`
- 데이터베이스/캐시: `MySQL`, `MongoDB`, `Redis`
- 분산 처리/스트리밍: `Apache Spark`, `Apache Kafka`, `Delta Lake`
- 클라우드/인프라: `AWS`, `S3`, `EKS`, `Ubuntu`, `Terraform`
- ML 운영: `scikit-learn`, `MLflow`, `FastAPI`
- 워크플로우 자동화: `Airflow`, `Docker`, `Kubernetes`, `Helm`

## 다른 과정과의 차별점

- 데이터 엔지니어링부터 MLOps 엔지니어 자동화까지 **한 과정 안에서 통합**합니다.
- `AI Ready Data`를 전면에 둬서, 단순 모델 운영이 아니라 AI 학습에 적합한 데이터 준비 역량까지 포함합니다.
- 실무 프로젝트에서 `CI/CD/CT`, 성능 평가 기반 자동 재학습, OCR/VOC 분석 모델 운영 같은 실제 운영 시나리오를 다룹니다.
- 데이터 기술과 운영 기술을 하나의 누적 포트폴리오로 만들 수 있다는 점이 강점입니다.

## 어떤 사람에게 잘 맞는가

- 비전공자라면: 알고리즘보다 시스템 흐름과 자동화 구조에 관심이 있는 사람
- 직무 전환자라면: 기존 산업 도메인의 데이터 흐름 이해를 AI 인프라 역량으로 확장하고 싶은 사람
- 개발 경험자라면: `Spark`, `Kafka`, `Kubernetes`, `Terraform`, `CI/CD/CT`까지 심화하고 싶은 사람
- 졸업 직후 진입자라면: 신입이어도 실무 투입 가능한 수준의 인프라/운영 포트폴리오를 만들고 싶은 사람

## 상담 시 바로 쓰기 좋은 문장

- **AI Ready 데이터 엔지니어링 캠프는 어떤 과정인가요?** AI 모델이 실제 서비스에서 멈추지 않고 안정적으로 돌아가게 만드는 기술을 배우는 과정입니다.
- **이 과정의 핵심은 무엇인가요?** 데이터 엔지니어링, AI Ready Data, 모델 운영 자동화가 핵심입니다.
- **어떤 사람에게 잘 맞나요?** 모델을 만드는 것보다 데이터 파이프라인, 배포, 운영 자동화, 모니터링에 더 관심 있는 사람에게 잘 맞습니다.
## 실제 응답 기준

- **AI Ready 데이터 엔지니어링 캠프의 포트폴리오는 몇 단계로 구성되나요?** 3단계(Lv.1~Lv.3)로 구성됩니다. Lv.1은 데이터 엔지니어링 실무 수준, Lv.2는 데이터 플랫폼 엔지니어 수준, Lv.3는 MLOps 엔지니어 수준입니다.
- **AI Ready 데이터 엔지니어링 캠프가 어떤 사람한테 잘 맞나요?** 모델 배포, 운영 자동화, CI/CD/CT, 모니터링 등 파이프라인 관점에 관심이 있는 사람, 데이터 파이프라인과 MLOps 엔지니어 플랫폼 구축 경험을 쌓고 싶은 사람에게 잘 맞습니다.
